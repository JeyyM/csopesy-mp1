CSOPESY OS MP — Codebase Reading & Commenting Order
====================================================

Read bottom-up by dependency, then trace one command through the system.


PHASE 0 — ORIENTATION (read, no commenting needed)
------------------------------------------------
  README.txt       Commands, ownership, how to run
  CMakeLists.txt   Which .cpp files exist and link together
  config.txt       Settings that drive runtime behavior


PHASE 1 — SMALL UTILITIES (read first, comment lightly)
-------------------------------------------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  1      src/TimeUtil.h                     Timestamps for logs and screen -ls
         src/TimeUtil.cpp

  2      src/OutputManager.h                outputs/ folder paths and clearing
         src/OutputManager.cpp

  Comment focus: what each helper returns, when it gets called.


PHASE 2 — CONFIGURATION
-----------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  3      src/Config.h                       config.txt parsing and validation
         src/Config.cpp

  Comment focus: each config field, valid ranges, which modules read it.


PHASE 3 — PROCESS DATA MODEL
----------------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  4      src/ProcessModel.h                 Process state: variables, instructions,
         src/ProcessModel.cpp               logs, status, formatSmi()

  Comment focus: ProcessStatus enum, what lives in memory vs what the scheduler
                 updates at runtime.


PHASE 4 — INSTRUCTION EXECUTION (no threading)
-----------------------------------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  5      src/InstructionEngine.h            Parsing and running ADD, PRINT, FOR,
         src/InstructionEngine.cpp            SLEEP, etc.

  Comment focus: parse -> execute flow, PRINT string concatenation, FOR handling.


PHASE 5 — SCHEDULER (core of the project)
-----------------------------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  6      src/Scheduler.h                    Public API, threads, queues, snapshots

  7      src/Scheduler.cpp                  Read internal sections in this order:
           1. start() / stop() / stopGracefully()     lifecycle
           2. addStandardProgram() / createProcessLocked()   process creation
           3. tickLoop() / advanceGlobalCpuCycles() / maybeSpawnBatchProcess()
           4. schedulerLoop() / coreLoop()            dispatch and CPU workers
           5. runProcessSlice() / runInstructionTree()   one quantum of work
           6. requeueProcess() / statusSnapshot()     RR requeue and reporting

  Comment focus: which thread runs what, readyQueue_ vs coreCurrent_, when batch
                 spawn stops after scheduler-stop.


PHASE 6 — CLI / PRESENTATION LAYER
----------------------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  8      src/ConsoleManager.h               Header, prompts, screen clear
         src/ConsoleManager.cpp

  9      src/ReportManager.h                screen -ls and report-util formatting
         src/ReportManager.cpp

  10     src/ScreenManager.h                  screen -s, screen -r, screen -ls,
         src/ScreenManager.cpp                process screen loop

  Comment focus: which CLI commands call which scheduler methods.


PHASE 7 — ENTRY POINT (tie it together last)
--------------------------------------------
  Order  Files                              What to understand
  -----  ---------------------------------  ----------------------------------
  11     src/main.cpp                       Main loop, init gate, command routing

  Comment focus: initialize gate, command -> module mapping, exit cleanup order
                 (scheduler.stop() then clear outputs).


ONE COMMAND TO TRACE AFTER READING
----------------------------------
Follow one path file-by-file:

  initialize
    main.cpp -> Config.cpp

  scheduler-start
    main.cpp -> Scheduler.cpp (start + batch generation)

  screen -s proc-01
    main.cpp -> ScreenManager.cpp -> Scheduler.cpp

  process-smi
    ScreenManager.cpp -> ProcessModel.cpp

  scheduler-stop
    main.cpp -> Scheduler.cpp (stopGracefully)

  exit
    main.cpp -> Scheduler.stop() -> OutputManager.clearAllProcessOutputs()


SUGGESTED COMMENTING STYLE
--------------------------
  - File header: 2-3 lines describing the module's role
  - Classes / key functions: what it owns, who calls it
  - Skip: obvious getters, empty lines, CMake/build files
  - Prioritize comments in: Scheduler.cpp, main.cpp, InstructionEngine.cpp
